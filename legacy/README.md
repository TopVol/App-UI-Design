# Legacy Recovery

Dieser Ordner dokumentiert den Wiederaufbau der ursprünglich hochgeladenen Repo-Artefakte, nachdem beim frühen Tree-Umbau Dateien aus dem sichtbaren Repo-Baum gefallen sind.

## Status

- Die neue modulare v4.9-Struktur bleibt aktiv.
- `Layout Rules.txt` wurde wieder ins Repo aufgenommen.
- Die folgende Liste dokumentiert die Originaldateien aus dem Upload, die als Legacy-/Quellreferenzen gelten.

## Ziel

Die Legacy-Dateien dienen als Referenz für:

1. Rückführung fortgeschrittener Constraint-Logik
2. Vergleich zwischen Monolith und modularer Struktur
3. spätere selektive Migration ohne Informationsverlust
